# 概要设计
## 分析方法
* 模块：基于系统需求的业务领域聚类

### HIPO
* [HIPO图](https://blog.csdn.net/wangjingna/article/details/41318739) = H图(层次结构图) + IPO图(输入、处理、输出)。H图是“模块地图”，IPO图是“模块的说明书”。
    * H图：是HIPO图的“宏观架构层”，自上而下展示系统的模块层级关系：系统总目标 → 顶层模块 → 子模块 → 不可再拆的原子模块(模块的功能)
    * IPO图(Input-Process-Output Chart)：是HIPO图的“微观细节层”，是H图中的“单个模块”（尤其是底层原子模块）的细节说明，有输入数据、核心处理逻辑、输出结果。

| 项 | 内容 | 示例(订单创建子模块) |
| - | - | - |
| 输入 | 模块运行所需的“外部数据/触发条件” | 1. 商品数据：商品 ID、单价、购买数量（来自商品管理模块）<br/> 2. 用户数据：用户 ID、收货地址、联系电话（来自用户管理模块）<br/> 3. 触发条件：用户点击 “提交订单” 按钮 |
| 处理 | 模块核心逻辑步骤(如何将输入转化为输出)，需细化到可执行的操作 | 1. 库存校验：调用商品模块的库存校验接口 <br/> 2. 金额计算：订单金额 =（单价 × 数量）+ 运费（根据收货地址计算）<br/> 3. 订单生成：按规则生成唯一订单号（格式：YYYYMMDD+6 位随机数）<br/> 4. 数据存储：将订单号、用户 ID、商品 ID、金额、状态（待支付）存入订单数据库表 |
| 输出 | 模块运行后的“结果数据/反馈” | 1. 成功输出：订单 ID（传递给支付模块）、订单状态（待支付，传递给用户模块用于 APP 展示）<br/> 2. 失败输出：错误提示（如 “库存不足”，反馈给用户模块用于页面显示） |
